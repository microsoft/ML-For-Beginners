# Machine Learning for Beginners - One Curriculum

> üåç Make we waka around di world as we dey explore Machine Learning through different world cultures üåç

Cloud Advocates for Microsoft dey happy to offer one 12-week, 26-lesson curriculum wey dey all about **Machine Learning**. For this curriculum, you go learn wetin dem dey call **classic machine learning**, wey dey mainly use Scikit-learn as library and we dey avoid deep learning (we don cover deep learning for our [AI for Beginners' curriculum](https://aka.ms/ai4beginners)). Pair these lessons with our ['Data Science for Beginners' curriculum](https://aka.ms/ds4beginners) too!

Travel with us around the world as we apply these classic techniques to data from many parts of the world. Each lesson get pre- and post-lesson quizzes, written steps to finish the lesson, solution, assignment, and more. Our project-based way of teaching make you learn as you build ‚Äî na proven way make new skills stick.

# How to Start

Follow these steps:
1. **Fork the Repository**: Click di "Fork" button wey dey top-right corner of this page.
2. **Clone the Repository**:   `git clone https://github.com/microsoft/ML-For-Beginners.git`

> [find all extra resources for this course for our Microsoft Learn collection](https://learn.microsoft.com/en-us/collections/qrqzamz1nn2wx3?WT.mc_id=academic-77952-bethanycheum)

> üîß **You need help?** Check our [Troubleshooting Guide](TROUBLESHOOTING.md) for solutions to common problems wey fit show for installation, setup, and when you dey run lessons.


**[Student dem](https://aka.ms/student-page)**, to use this curriculum, fork the whole repo to your own GitHub account and finish the exercises by yourself or with group:

- Start with a pre-lecture quiz.
- Read the lecture and finish the activities, pause and reason on every knowledge check.
- Try build the projects by understanding the lessons instead of just running the solution code; but the solution code dey inside the `/solution` folders for each project lesson.
- Do the post-lecture quiz.
- Finish the challenge.
- Finish the assignment.
- After you don finish one lesson group, go the [Discussion Board](https://github.com/microsoft/ML-For-Beginners/discussions) and "learn out loud" by filling the correct PAT rubric. 'PAT' mean Progress Assessment Tool ‚Äî na rubric wey you go fill to help your learning. You fit still react to other PATs so we fit learn together.

> For more study, we recommend make you follow these [Microsoft Learn](https://docs.microsoft.com/en-us/users/jenlooper-2911/collections/k7o7tg1gp306q4?WT.mc_id=academic-77952-leestott) modules and learning paths.

**Teachers dem**, we don [include some suggestions](for-teachers.md) on how you fit use this curriculum.

## How we dey teach

We choose two teaching principles when we build this curriculum: make am hands-on and **project-based**, and make sure e get **frequent quizzes**. Plus, this curriculum get one common **theme** to make everything join well.

Because the content dey match projects, the process dey more interesting for students and e go help concepts stick more. Also, small quiz before class go set student mind to learn the topic, and quiz after class go help make the knowledge remain. This curriculum design flexible and fun ‚Äî you fit take am full or small-small. Projects start small and dem go dey more complex by the end of the 12-week cycle. This curriculum still get one postscript wey dey talk how ML dey work for real world ‚Äî you fit use am as extra credit or for discussion.

> Find our [Code of Conduct](CODE_OF_CONDUCT.md), [Contributing](CONTRIBUTING.md), [Translation](TRANSLATIONS.md), and [Troubleshooting](TROUBLESHOOTING.md) guidelines. We dey welcome your constructive feedback!

## Wetin each lesson dey include

- optional sketchnote
- optional supplemental video
- video walkthrough (some lessons only)
- [pre-lecture warmup quiz](https://ff-quizzes.netlify.app/en/ml/)
- written lesson
- for project-based lessons, step-by-step guides on how to build the project
- knowledge checks
- a challenge
- supplemental reading
- assignment
- [post-lecture quiz](https://ff-quizzes.netlify.app/en/ml/)

> **One small tori about languages**: These lessons mostly dey written for Python, but plenty dem still dey available for R. To finish R lesson, go the `/solution` folder and find R lessons. Dem get .rmd extension wey mean **R Markdown** file ‚Äî na file wey dey embed `code chunks` (R or other languages) and get `YAML header` (wey dey guide how to format outputs like PDF) inside one `Markdown document`. So, e good as authoring framework for data science because e allow you combine your code, the output, and your notes by writing dem down for Markdown. R Markdown documents fit render to formats like PDF, HTML, or Word.

> **One small tori about quizzes**: All quizzes dey inside the [Quiz App folder](../../quiz-app), total na 52 quizzes, each get three questions. Dem link from the lessons but you fit still run the quiz app locally; follow the instruction for the `quiz-app` folder to host locally or deploy to Azure.
